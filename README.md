# Monkey-Motor-Neural-Forecasting-Model-NSF-HDR
This repo contains our solution to the Neural Forecasting problem in the UCSD SMASH &amp; NSF HDR Hackathon held at UC San Diego on January 17-18, 2026, which won 1st place and is submitted as "Torch Tritons" https://www.codabench.org/competitions/9806/#/pages-tab

Our team consisted of myself along with [Aidan Manternach](https://www.linkedin.com/in/aidan-manternach/?lipi=urn%3Ali%3Apage%3Ad_flagship3_detail_base%3BnaNSJT1GS4%2BAcXkXecTuDg%3D%3D), [Jaisurya Sundarbabu](https://www.linkedin.com/in/jaisurya-sundarbabu/?lipi=urn%3Ali%3Apage%3Ad_flagship3_detail_base%3BnaNSJT1GS4%2BAcXkXecTuDg%3D%3D), [Manasvin Surya B J](https://www.linkedin.com/in/manasvin-surya/?lipi=urn%3Ali%3Apage%3Ad_flagship3_detail_base%3BnaNSJT1GS4%2BAcXkXecTuDg%3D%3D), and [Dmitri Singer](https://www.linkedin.com/in/dmitrisinger/?lipi=urn%3Ali%3Apage%3Ad_flagship3_detail_base%3BnaNSJT1GS4%2BAcXkXecTuDg%3D%3D).

The data consists of neural activity samples from two monkeys ("Affi" and "Beignet") stored as 4D tensors, containing neuron count, signal strength per neuron, time, and a composite 9-signal "feature" that the model predicts, which is the target variable. Our objective was to build a model that, given the first 10 seconds of any 20-second "feature", could accurately predict the remainder of the signal. We used a Graph Neural Network + Transformer architecture for our solution; instead of assuming fixed neuron connections, we let the model learn an adaptive graph to discover which electrodes actually influence each other. The transformer is a temporal transformer on top of spatial features that captures long-range dynamics and how patterns evolve over time. Unlike same session modeling, the task required handling session-to-session drift, changing electrode behavior, and incomplete observations, making generalization difficult.
